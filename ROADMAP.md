# Roadmap

This document outlines the planned features and improvements for gettextify.

## Version 1.0.0 (Current)

- [x] CSV to MO conversion
- [x] Modular architecture
- [x] Custom CSV delimiters
- [x] UTF-8 support
- [x] Basic CLI interface
- [x] Meson build system

## Version 1.1.0 (Planned)

### Input Formats

- [ ] PO (Portable Object) file parser
- [ ] POT (Portable Object Template) file parser
- [ ] JSON translation format support
- [ ] XML (XLIFF) format support

### Output Formats

- [ ] PO file writer (MO decompiler)
- [ ] JSON output format
- [ ] CSV export from MO files

### Error Handling

- [ ] Better error messages with line numbers
- [ ] Validation mode (check translations without compiling)
- [ ] Warning system for potential issues
- [ ] Strict mode for production builds

## Version 1.2.0 (Planned)

### Features

- [ ] Plural forms support
- [ ] Context support (msgctxt)
- [ ] Multiple input files merging
- [ ] Fuzzy translation handling
- [ ] Translation statistics output

### Quality Improvements

- [ ] Unit test suite
- [ ] Integration tests
- [ ] Continuous Integration setup
- [ ] Code coverage reports
- [ ] Static analysis integration

## Version 2.0.0 (Future)

### Advanced Features

- [ ] Interactive mode
- [ ] Plugin system for custom formats
- [ ] Translation memory integration
- [ ] Automatic encoding detection
- [ ] Batch processing mode

### Performance

- [ ] Parallel processing for large files
- [ ] Memory optimization for huge catalogs
- [ ] Incremental compilation
- [ ] Caching system

### Developer Experience

- [ ] Library mode (libgettextify)
- [ ] C API for language bindings
- [ ] Python bindings
- [ ] JavaScript/TypeScript bindings
- [ ] Documentation improvements
- [ ] API reference generation

## Long-term Goals

### Tooling Ecosystem

- [ ] Web-based translation editor
- [ ] Translation validation tools
- [ ] Format conversion utilities
- [ ] Translation workflow automation
- [ ] Integration with popular frameworks

### Standards Compliance

- [ ] Full GNU gettext compatibility
- [ ] XLIFF 2.0 support

### Quality Assurance

- [ ] Translation quality checks
- [ ] Consistency validation
- [ ] Terminology enforcement
- [ ] Style guide compliance
